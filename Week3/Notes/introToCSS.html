Intro to CSS

So now we have a good understanding of HTML, but the last page still looked pretty ugly. The browser applies some default styling with certain tags, like holding <strong> tags, but we were still looking at a very basic document - no colors, no other fonts, no spacing, etc. So how do we actually style the page to make it look nicer? We use CSS!

CSS stands for Cascading Style Sheets. It is a way of styling html elements in many different ways. It follows certain rules that we define. There are three ways to include CSS in an html page.

  1. Inline
  2. Using internal stylesheets
  3. Using external stylesheets
What we saw in the previous introToHTML.html file was an example of inline styling. It can be performed on any html element with the style attribute. Within the quotations, we define the value of the style using CSS rules. CSS rules are key-value pairs separated by a colon ( : ). There are many CSS rules that you can apply, we will go over a few, but further self-exploration is required. We'll start with the color rule, which will change text color.

Internal Stylesheets

We can declare a style tag in the head tag of this html document. It will declare some overall styles for this entire html document.

If you look above, you'll see in the head of the document, a style tag. This will declare our styling to whatever elements are declared in the style tag. This is called a stylesheet. We use CSS selectors followed by a curly brace with multiple styling rules separated by semicolons. The CSS selector is a name of a tag, for example, the body tag. Note that this will affect ALL elements that are children of that tag as well. So the first CSS rule above applied to the body tag restricts the width of the body to only 800 pixels at a maximum and sets the margin to be automatically calculated, which centers the element within it's parent. This is why our page looks like one single column down the middle. The final rule specifies the font family as Helvetica, but with multiple provided with a comma. So that if the browser does not support one, it can use the next. Here, we prefer to use Helvetica and will fall back to Arial. As we saw earlier, CSS comments use the /* and */ similar to Java's multi-line comments.
